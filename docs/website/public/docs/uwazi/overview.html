Overview

The Uwazi API empowers insurance providers by streamlining the verification of hospital insurance claims. It determines whether treatment procedures are under-reported, within policy limits, or exceed the allowable limits. By providing comprehensive insights, the API equips insurers with the critical information needed to make well-informed decisions when approving claims.

Setup​

In order to run the application, you will need to have the following software installed on your machine:

Docker Desktop​

Docker Desktop is a tool for building, running, and managing containers. It provides a graphical user interface (GUI) for managing Docker containers and images.

To install Docker Desktop, follow the instructions provided on the Docker website.

Once you have Docker Desktop installed, you can proceed to open the docker desktop by cliicking on the Docker icon in the system tray. To verify that Docker is installed and running, you can run the following command in your terminal:

docker --version

Git​

Git is a distributed version control system that allows you to track changes in your codebase. To install Git, follow the instructions provided on the Git website.

After git is installed, you can clone the repository onto your local machine using the repository's githb url. In order to get the github url of the repository, you can click on the code button and copy the url as shown in the image below:

You can then run the following git command to clone the repository

git clone <github-url>

mkcert​

mkcert is a tool for generating locally trusted development certificates. It allows you to create a local development certificate that can be used to secure your development environment. It is useful for developing web applications that require secure communication over HTTPS and helps in avoiding certificate errors in your development environment. It also assists in providing a consistent development experience that matches production.

To install mkcert follow the instructions in the mkcert GitHub repository. After installation, generate certificates by running the following commands in your terminal.

mkcert -install
mkcert localhost

Copy the generated certificates which should be localhost-key.pem and localhost.pem to the config/nginx/certs directory.

Mailing Configuration​

The application uses Gmail SMTP to send emails. This requires setting up a gmail app and its associated app password which is separate from the gmail account password.

To set up the Gmail SMTP configuration, visit Google App Passwords and add a new app then copy the generated password as you wont be able to see it again once you leave the page or close the modal.

The app password is the password you will use to configure the mailer while the gmail email account is the email address you will use to send emails via the mailer. These settings are configured in the .env file in the root directory of the project as follows:

  • MAIL_USERNAME - set to the gmail email account.
  • MAIL_PASSWORD - set to the gmail app password.
